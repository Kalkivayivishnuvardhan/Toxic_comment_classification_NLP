How It Works

From raw comment to toxicity verdict in milliseconds — a battle-tested NLP pipeline.

01 — INPUT
📝

Text Preprocessing

Lowercasing, URL removal, HTML stripping, punctuation cleanup, stop-word filtering, and lemmatization produce a clean normalized text representation.

02 — ENCODE
🔢

Feature Extraction

TF-IDF vectorization for classical models, Keras sequence encoding for BiLSTM, and WordPiece/BPE tokenization for BERT and RoBERTa transformers.

03 — PREDICT
🎯

Multi-Label Inference

Each of the 6 toxicity labels gets an independent sigmoid probability. Labels exceeding the configurable threshold (default 0.5) are flagged.

The Jigsaw Toxic Comment Classification Challenge dataset from Kaggle — 159,571 Wikipedia Talk Page comments labeled across six toxicity categories. It's one of the most widely used benchmarks for content moderation research.
The pipeline illustrates a progression from interpretable classical ML (Logistic Regression, XGBoost) through deep learning (BiLSTM) to state-of-the-art transformers (BERT, RoBERTa). Each offers a different latency/accuracy tradeoff for production deployment decisions.
Each label receives a sigmoid probability (0–1). The threshold determines the cutoff above which a label is considered positive. Lowering it increases recall (catches more toxic content) at the cost of more false positives. Default is 0.5.
All models were trained on the same cleaned text (URL removal, HTML stripping, lowercasing, stop-word filtering, lemmatization). Using the exact same preprocessing at inference time prevents train-serving skew that would degrade accuracy in production.
Logistic Regression uses class_weight='balanced', XGBoost uses scale_pos_weight, and BERT/RoBERTa use weighted sampling during fine-tuning. The label distribution is heavily skewed (e.g., only ~1% of comments are threats), so these adjustments are critical.
